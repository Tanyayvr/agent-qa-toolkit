Agent QA Toolkit
Portable Evidence Packs for tool-using AI agents
- Offline, self-hosted, no data egress
- CI gate decisions + security signals
- Target: platform/QA engineers shipping agents in production

Problem
- Incident handoff is still screenshots + partial logs
- Sharing runs outside your tracing UI is slow or impossible
- Tool I/O and retrieval context are scattered
- Secrets/PII make sharing risky

Solution
- One run → one portable Evidence Pack
- report.html (human) + compare-report.json (CI contract)
- assets/ + manifest.json with sha256 integrity
- Strict verify: portability + integrity checks
- NOW: strict-signature parity checks across Node/Python/Go validators (signed pack must pass, tampered manifest must fail)
- NEW: Historical Trending (local SQLite) to catch regression and cost drift over time
- NEW: data-driven timeout onboarding profile (calibration -> validation -> production) for slow external agents
- NEW: telemetry normalization levels (native -> inferred from stdout -> wrapper fallback) to avoid "silent no-trace" runs
- NEW: release-gate E2E in CI (policy-gate + soak/load stability checks)
- NOW: runtime policy hardening shipped (planning/repl policy enforcement in adapter + deterministic evaluator checks)
- NOW: quality profile requires strong tool telemetry (`native|inferred`; wrapper-only is treated as insufficient evidence)
- NOW: plugin adapters fail fast on telemetry mismatch (`invalid_telemetry`) when tool activity is hinted but structured tool events are missing
- NOW: semantic quality checks shipped for text tasks (`required/forbidden concepts` + calibrated profile/thresholds + custom synonyms)
- NOW: release campaign gate is ON by default and can enforce KPI thresholds (`pre-action entropy removed`, `reconstruction min saved/block`)
- NOW: quality gate includes high-confidence repository secret scan (keys/tokens/private-key markers)
- NOW: auto-timeout guardrails shipped (success-only learning, min successful samples, bounded growth vs base timeout)
- NOW: campaign runs are ingested into local evidence library by default (opt-out flag only)

How It Works
- Runner executes cases against the agent
- Evaluator diffs baseline vs new
- Outputs Evidence Pack (offline, shareable)
- CI consumes per-case gate decisions
- Timeout auto-profile learns from observed run history instead of fixed timeout guesses

What You Get
- Regression diffs per case
- Root cause attribution (RCA)
- Security scanners (PII, injection, exfiltration, risk)
- Self-contained, portable report directory
