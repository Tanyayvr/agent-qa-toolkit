Agent QA Toolkit
Portable Evidence Packs для tool-using AI агентов
- Offline, self-hosted, без утечки данных
- CI gate decisions + security сигналы
- ЦА: platform/QA инженеры, продакшн-агенты

Проблема
- Хэндовер инцидентов = скриншоты + частичные логи
- Передать run без доступа к UI сложно
- Tool I/O и retrieval контекст разрознены
- Секреты/PII делают шаринг рискованным

Решение
- Один run → один Evidence Pack
- report.html (человеку) + compare-report.json (CI контракт)
- assets/ + manifest.json с sha256 целостностью
- Strict verify: portability + integrity checks
- НОВОЕ: Historical Trending (локальная SQLite) для дрейфа регрессий и костов во времени

Как работает
- Runner запускает кейсы
- Evaluator сравнивает baseline vs new
- Выдаёт Evidence Pack (offline, shareable)
- CI читает gate decision per-case

Что даёт
- Regression diffs по кейсам
- Root cause attribution (RCA)
- Security scanners (PII, injection, exfiltration, risk)
- Self-contained, portable report directory

Historical Trending (Pro+ add‑on, когда доступен)
- Дрейф регрессий между релизами (не один дифф)
- Flaky‑кейсы: pass/fail во времени
- Тренд gate‑решений: none/approval/block
- Тренд токен‑костов: ранний сигнал удорожания
- Release safety: “зелёное сегодня, сломалось завтра” становится видимым
- Разбор инцидентов: когда началась деградация
- Offline‑first: SQLite + HTML, без egress
- Shareable: trend.html можно вложить в тикет
- Без зависимости от tracing UI: тренды из Evidence Packs
- Vendor‑neutral: работает в air‑gapped
- CI‑friendly CLI: trend runs/tokens/html
- Audit readiness: исторические доказательства для compliance
- Измерение улучшений по релизам
- Контроль затрат: база для бюджетов/лимитов
- Без скрытой телеметрии

Historical Trending (offline)
- Каждый run пишется в локальную SQLite базу
- Trend CLI + trend.html (без SaaS и внешних запросов)
- Тренды регрессий, gate-решений и токен‑костов

Почему не Promptfoo / DeepEval / LangSmith / Galileo
- Они сильны в model evals и/или observability
- Мы про agent behavior + переносимый offline handoff
- Evidence Pack можно вложить в тикет/PR/incident-док
- Детерминированный CI gate из одного машинного контракта (`compare-report.json`)
- Путь комплементарности: оставить observability платформу и добавить нас как release-evidence слой в CI
- Плюс: локальная БД трендов и офлайн‑графики (без lock‑in и egress)

Деплой
- Только self-hosted
- Работает в air-gapped
- Нет аккаунтов и внешнего хранения
- Вы контролируете ретеншн и шаринг
- Trend DB локальная (SQLite), без SaaS/телеметрии
