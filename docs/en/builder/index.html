EU AI Act Annex IV technical documentation

Write short factual descriptions. Use 1-3 short sentences per field and describe the real system in plain language.

1. Identify the system

Start with the basic facts that explain what the system is for and who provides it.

Law source: Annex IV(1)(a)

State what the system is used for, who uses it, and what task it supports.

Law source: Annex IV(1)(a)

State the AI provider name and the current system or agent version covered by this draft.

2. Describe interfaces and delivery

Explain what the system connects to and how it is delivered or put into use.

Law source: Annex IV(1)(b)

List the main systems, tools, APIs, databases, or hardware this system depends on or exchanges data with.

Law source: Annex IV(1)(c)

Describe the main parts of the system and how the current version was built, tested, and released.

Law source: Annex IV(1)(d)

State how the system is made available or put into use, for example as SaaS, API, internal tool, or embedded feature.

Law source: Annex IV(1)(e)

State the hardware or infrastructure the system is expected to run on.

3. Explain use and outputs

Describe what users see, what they are told to do, and what output the system produces.

Law source: Annex IV(1)(g)

Describe the interface a deployer or operator sees when using the system.

Law source: Annex IV(1)(h)

Summarize the instructions, limits, and expected use conditions given to the people who use the system.

Law source: Annex IV(2)(b)

State what output the system produces and what level or quality of that output you expect.

4. Describe monitoring and testing

Finish the section with the controls that watch the system and the checks that validate it.

Law source: Annex IV(2)(f)

Describe how the system is monitored, what alerts or controls exist, and how issues can be paused or escalated.

Law source: Annex IV(2)(g)

Describe the main validation or testing procedures you run and the metrics or acceptance checks you use.
